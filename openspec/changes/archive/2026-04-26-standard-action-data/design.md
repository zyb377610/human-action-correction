## Context

Phase 1 的 PoseEstimator 已能提取关键点，PoseSequence 支持 .npy/.json 序列化。本模块在此基础上构建标准动作数据管理体系。

### 数据流图

```
┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 摄像头/  │───▶│ PoseEstimator│───▶│ 预处理流水线  │───▶│ 标准模板库   │

                                           │

                                    │ 拉伸/噪声  │
                                    └─────────────┘
```

## Goals / Non-Goals

**Goals:**
- 提供录制标准动作的交互式工具
- 实现序列预处理（滤波、插值、归一化到固定长度）
- 建立基于文件系统的动作模板库（每类动作一个子目录）
- 支持数据增强扩充训练样本

**Non-Goals:**
- 不实现数据库存储（文件系统足够）
- 不实现在线标注平台
- 不实现自动动作分割（手动指定起止）

## Decisions

### D1: 模板库结构
```
data/templates/
├── squat/              # 深蹲
│   ├── standard_01.json
│   └── standard_02.json
├── pushup/             # 俯卧撑
│   └── standard_01.json
└── metadata.json       # 全局元信息（动作类别列表）
```

### D2: 预处理流水线顺序
缺失值插值 → 平滑滤波(Savitzky-Golay) → 序列长度归一化(重采样到固定帧数)

### D3: 模块文件组织
```
src/data/
├── __init__.py
├── preprocessing.py    # 滤波、插值、归一化
├── template_library.py # 模板库管理
├── augmentation.py     # 数据增强
└── recorder.py         # 录制工具
```

## Risks / Trade-offs
- **样本量有限**：毕业设计场景下标准动作样本少，通过数据增强缓解
- **动作类别选择**：初期聚焦 3-5 种简单动作（深蹲、举臂、侧弯等）
