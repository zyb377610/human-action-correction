## Context

当前系统已完成 7 个阶段的开发（姿态估计 → 数据预处理 → DTW 对比 → 深度学习分类 → 矫正反馈 → Web UI → 系统集成），但实时模式仅支持「录制完成后离线分析」，无法满足论文要求的 ≤300ms 实时反馈。

现有架构中可复用的模块：
- `AngleCalculator`：计算关节角度（~2ms）
- `CorrectionRuleEngine`：40 条专家规则，根据角度偏差生成文字建议（~2ms）
- `PoseEstimator`：MediaPipe 单帧推理（~30-50ms CPU）
- `TemplateLibrary`：标准动作模板（含逐帧关节角度参考值）
- `ActionComparator`：DTW 对比器，已支持 DTW / FastDTW / DDTW 三种算法

## Goals / Non-Goals

**Goals:**
- 实时模式：选择动作 → 3秒倒计时 → 跟做时每帧用固定窗口 DTW + 角度对比给出即时矫正建议（延迟 ≤300ms）→ 结束后总体评分
- 视频分析 Tab 新增摄像头录制输入方式
- UI 中新增 DTW 算法选择下拉框（DTW / FastDTW / DDTW / 自动），方便用户测试对比
- 复用现有模块，不新增外部依赖

**Non-Goals:**
- 不改变 DTW 算法实现本身（复用已有的三种算法）
- 不训练新模型
- 不做移动端适配

## Decisions

### 决策 1：实时反馈策略 — 固定窗口 DTW + 角度对比

**选择：固定窗口 DTW（最近 10 帧）**

| 方案 | 每帧耗时 | 精度 | 复杂度 |
|------|---------|------|--------|
| 单帧角度对比 | ~2-5ms | 中等 | 低 |
| **固定窗口 DTW（10帧）** | **~5-10ms** | **较高** | **中** |
| 累积序列 DTW | 50ms→500ms+ | 最高 | 高 |

理由：固定窗口 DTW 取最近 10 帧与模板对应位置的 10 帧做对比，计算量恒定（10×10=100），耗时稳定在 ~5-10ms，远在 300ms 预算内。同时结合角度偏差对比给出具体矫正文字。

### 决策 2：模板进度同步方式

**选择：基于时间比例的帧映射**

```
user_progress = current_frame_index / expected_total_frames
template_frame_index = int(user_progress * template.num_frames)
```

窗口映射：取模板的 [template_frame_index-5, template_frame_index+5] 范围的帧段。

### 决策 3：DTW 算法选择 — 四种模式

UI 中提供算法选择下拉框，影响所有分析流程：

| 选项 | 离线分析算法 | 实时窗口算法 | 适用场景 |
|------|------------|------------|---------|
| 经典 DTW | `dtw` | `dtw` | 基准对比 |
| FastDTW | `fastdtw` | `fastdtw` | 速度优先 |
| DerivativeDTW | `ddtw` | `ddtw` | 形状匹配 |
| 自动选择 | `ddtw` | `fastdtw` | 推荐日常使用 |

算法选择通过 `AppPipeline` 传递给 `ActionComparator` 和 `RealtimeFeedbackEngine`，无需修改底层算法代码。

### 决策 5：视频分析 Tab — 摄像头录制 + 算法选择

- 模式 A：上传本地视频（现有功能，不变）
- 模式 B：摄像头录制（录制完成后走与上传视频相同的分析流程）
- 算法下拉框同样出现在视频分析 Tab 中

## Risks / Trade-offs

- **[风险] 模板进度同步不准确** → 用户跟做速度与模板不一致时，窗口对比的参考帧段可能错位。缓解方案：窗口取 ±5 帧范围，有一定容错空间。
- **[风险] Gradio streaming 延迟波动** → 网络延迟可能导致部分帧超过 300ms。缓解方案：实测如果稳定在 200ms 左右则可接受。
- **[权衡] 固定窗口 vs 累积序列** → 固定窗口牺牲了全局视角，但保证了恒定的计算量和延迟，通过录制结束后的完整 DTW 弥补。
- **[权衡] 四种算法选项增加 UI 复杂度** → 但提供了测试灵活性和论文数据采集能力，值得。
