## 三、功能 → 模块 → 技术选择 的完整映射

### 功能 1：「看到你的骨骼」→ Phase 1 姿态估计

**要实现的功能：**
系统需要"看懂"人体——知道你的肩膀在哪、膝盖弯了多少度。

**技术方案：**
用 MediaPipe 从摄像头画面中提取 33 个人体骨骼关键点的坐标。

**为什么选 MediaPipe（而不是 OpenPose、AlphaPose 等）：**

| 维度 | MediaPipe | OpenPose | AlphaPose |
|------|-----------|----------|-----------|
| 安装复杂度 | pip install 即可 | 需要编译 Caffe/CMake | 需要 GPU 环境 |
| 关键点数量 | 33 个全身 | 25 个(BODY_25) | 17 个(COCO) |
| CPU 实时性 | ~30FPS ✅ | 需要 GPU | 需要 GPU |
| 3D 支持 | 原生支持 x,y,z | 仅 2D | 仅 2D |
| 毕设演示友好度 | 任何电脑可运行 ✅ | 需要高配电脑 | 需要高配电脑 |

> 📚 参考依据：ASD 运动分析综述论文中统计了 16 篇基于骨骼的研究，其中 OpenPose 被 10 篇使用，
> 但 MediaPipe 在近年研究中快速增长（论文 Section VI.B），且在手部研究中被多篇采用。
> MediaPipe 论文本身是本项目的参考文献之一。

**代码模块如何对应到这个功能：**

```
用户打开摄像头
  ↓
video_source.py     → CameraSource 读取摄像头画面（每秒30帧图像）
  ↓
estimator.py        → MediaPipe 模型分析每帧图像，提取33个关键点坐标
  ↓
data_types.py       → PoseFrame 对象存储关键点数据（坐标+可见度）
  ↓
feature_extractor.py → 从原始坐标计算出"关节角度""骨骼比例""运动速度"等特征
  ↓
visualizer.py       → 在摄像头画面上绘制骨骼连线图
```

### 功能 2：「知道标准动作长什么样」→ Phase 2 标准动作数据

**要实现的功能：**
系统需要有一个"标准答案"——正确的深蹲、俯卧撑长什么样。

**技术方案：**
用 Phase 1 的工具录制标准动作视频，提取关键点序列，经过预处理后存为模板。

**为什么需要预处理（四个具体问题和对策）：**

| 问题 | 原因 | 解决方案 | 对应函数 |
|------|------|---------|---------|
| 关键点抖动 | 摄像头噪声、光线变化 | Savitzky-Golay 平滑滤波 | `smooth_sequence()` |
| 关键点缺失 | 身体部分被遮挡 | 线性插值填充 | `interpolate_missing()` |
| 动作快慢不同 | 每个人做动作的速度不同 | 重采样到统一帧数 | `resample_sequence()` |
| 训练样本太少 | 毕设场景数据有限 | 镜像翻转+噪声+时间拉伸 | `augmentation.py` |

**代码模块如何对应到这个功能：**

```
录制一段标准深蹲视频
  ↓
recorder.py          → 交互式录制（倒计时 → 录制 → 预览 → 保存）
  ↓
preprocessing.py     → 插值填充 → 平滑滤波 → 归一化到固定帧数
  ↓
template_library.py  → 存入 data/templates/squat/standard_01.json
  ↓
augmentation.py      → 镜像翻转生成 standard_01_mirror.json
                     → 添加噪声生成 standard_01_noisy.json
                     → 时间拉伸生成 standard_01_slow.json
                     → (1个录制样本 → 4个训练样本)
```

### 功能 3：「对比你和标准的差异」→ Phase 3 DTW 动作对比

**要实现的功能：**
把你实时做的动作和标准模板进行逐帧对比，给出相似度评分。

**技术方案：**
使用 DTW（Dynamic Time Warping，动态时间规整）算法。

**为什么选 DTW（而不是直接逐帧对比）：**

你做深蹲可能比标准快或慢。如果逐帧对比：

```
你的深蹲(慢):  站 → 蹲 → 蹲 → 蹲 → 起 → 起 → 站     (7帧)
标准深蹲(快):  站 → 蹲 → 蹲 → 起 → 站                 (5帧)

逐帧硬对比:    站↔站  蹲↔蹲  蹲↔蹲  蹲↔起  起↔站  起↔?  站↔?
                ✓     ✓     ✓     ✗     ✗    无法对齐！
```

DTW 能自动找到最佳对齐方式：

```
你的深蹲:      站 → 蹲 → 蹲 → 蹲 → 起 → 起 → 站
                ↕     ↕         ↕     ↕         ↕
标准深蹲:      站 → 蹲 → ---- → 蹲 → 起 → ---- → 站
               (DTW 自动拉伸标准动作来匹配你的速度)
```

> 📚 参考依据：DTW 综述论文全面分析了各种 DTW 变体。我们计划实现并对比：
> - **经典 DTW** (O(n²))：精确对比，适合离线分析
> - **FastDTW** (线性时间)：近似但速度快，适合实时场景（Salvador & Chan, 2007）
> - **DerivativeDTW**：不比坐标值，比运动趋势，对体型差异更鲁棒（Keogh & Pazzani, 2001）
> 这三种变体的实现与对比可以作为论文的技术贡献点。

### 功能 4：「识别你在做什么动作」→ Phase 4 深度学习分类

**要实现的功能：**
自动识别用户正在做的是哪种动作（深蹲？俯卧撑？举臂？），无需用户手动选择。

**技术方案：**
用 CNN-LSTM 深度学习模型，从关键点序列中学习动作特征并分类。

**为什么选 CNN-LSTM 架构：**

```
单帧数据（某一瞬间的姿态）        多帧数据（一段动作的变化过程）
┌─────────────────┐              ┌─────────────────────────┐
│ 手在头顶上方     │              │ 手从下往上移动           │
│ 膝盖完全伸直     │              │ 膝盖从弯到直            │
│ → 空间特征      │              │ → 时间特征              │
│ → CNN 擅长提取  │              │ → LSTM 擅长提取         │
└─────────────────┘              └─────────────────────────┘

CNN-LSTM 组合：
帧1 → CNN → 空间特征1 ─┐
帧2 → CNN → 空间特征2 ─┤
帧3 → CNN → 空间特征3 ─┼→ LSTM → 时序建模 → 分类: "深蹲"
帧4 → CNN → 空间特征4 ─┤
帧5 → CNN → 空间特征5 ─┘
```

> 📚 参考依据：ASD 运动分析综述 Table 6 统计了33篇文献的分类方法，
> 其中 CNN 被 8 篇使用，LSTM 被 8 篇使用，CNN-LSTM 组合在多项研究中
> 表现最佳（论文 Section V.C.1 "Spatiotemporal Hybrid Models"）。

**与 Phase 3 DTW 的互补关系：**

```
Phase 4 (深度学习):  回答"你在做什么动作？"  → 动作类别（深蹲/俯卧撑/...）
Phase 3 (DTW):      回答"你做得像不像标准？"  → 相似度评分（0-100分）

两者组合工作流:
  用户做动作 → Phase 4 识别"这是深蹲" → 自动选取深蹲的标准模板 → Phase 3 DTW 精确对比
```

### 功能 5：「告诉你怎么改」→ Phase 5 矫正反馈

**要实现的功能：**
不只是说"你做得不好"，而是具体指出哪个关节偏了、偏了多少、应该怎么调整。

**技术方案：**
基于 DTW 的逐关节对齐结果，定位偏差最大的关节，生成自然语言的矫正建议。

**技术逻辑：**

```
DTW 对齐后，可以逐关节对比每个时刻的角度差异：

           时刻1   时刻2   时刻3(最低点)   时刻4
你的左膝:   120°    115°    95°            100°
标准左膝:   120°    110°    75°            80°
偏差:         0°      5°    20° ← 最大！     20°

系统分析: 左膝在深蹲最低点偏差最大(20°)
  ↓
生成建议: "深蹲最低点时，左膝弯曲不够。
          当前角度95°，标准应为75°左右。
          请尝试再下蹲一些，保持膝盖与脚尖方向一致。"
```

**偏差严重程度分级：**

| 偏差范围 | 等级 | 反馈方式 |
|---------|------|---------|
| < 10° | ✅ 合格 | 绿色标注，无提示 |
| 10°-20° | ⚠️ 轻微偏差 | 黄色标注 + 简短提示 |
| > 20° | ❌ 明显偏差 | 红色标注 + 详细矫正建议 |

### 功能 6 & 7：「用户界面」和「系统集成」→ Phase 6 & 7

**Phase 6 前端界面：** 把上面所有能力包装成一个 Web 应用。使用 Streamlit 或 Gradio 框架，
用户打开浏览器就能使用，无需安装任何软件。

**Phase 7 系统集成：** 把所有模块串成一条完整的实时流水线：

```
摄像头画面(30FPS)
  → Phase 1: 姿态估计(提取关键点)
  → Phase 4: 深度学习(识别动作类别)
  → Phase 2: 加载对应标准模板
  → Phase 3: DTW 对比(计算相似度)
  → Phase 5: 矫正反馈(生成建议)
  → Phase 6: UI 展示(显示给用户)
  → 整个过程 < 100ms，实现实时反馈
```
