## 开发阶段与模块规划

### Phase 1: 姿态估计模块 ✅ 已完成规划

- **OpenSpec Change**: `pose-estimation-module`（已创建，含完整 proposal/design/specs/tasks）
- **目录**: `src/pose_estimation/`
- **核心能力**:
  - MediaPipe Pose 封装（33个关键点提取）
  - 视频输入管理（摄像头 + 文件）
  - 特征工程（关节角度、骨骼比例、运动速度）
  - 骨骼可视化
- **预计工时**: 8-12 小时
- **交付物**: 可运行的实时骨骼标注 Demo
- **下游依赖**: Phase 2, 3, 4, 5 均依赖本模块输出

---

### Phase 2: 标准动作数据采集与预处理 ✅ 已完成

- **OpenSpec Change**: `standard-action-data`（待创建）
- **目录**: `src/utils/`, `data/`, `scripts/`
- **核心能力**:
  - 标准动作录制工具（使用 Phase 1 的姿态估计录制关键点序列）
  - 数据标注规范（动作类别、起止帧、质量评级）
  - 数据预处理流水线（平滑滤波、缺失值插值、归一化）
  - 标准动作模板库（每个动作类别的"标准"关键点序列）
  - 数据增强（时间拉伸、噪声添加、镜像翻转）
- **预计工时**: 6-8 小时
- **参考文献支撑**:
  - ASD综述：数据采集方法、预处理流程、数据增强策略
- **前置依赖**: Phase 1（需要姿态估计模块提取关键点）

---

### Phase 3: DTW 动作对比模块 ✅ 已完成

- **OpenSpec Change**: `dtw-action-comparison`（待创建）
- **目录**: `src/action_comparison/`
- **核心能力**:
  - 经典 DTW 算法实现（基于 dtw-python 库）
  - FastDTW 近似算法（线性时间复杂度，适合实时场景）
  - Derivative DTW（基于一阶导数的形状匹配，参考 DTW 综述论文）
  - Sakoe-Chiba 带宽约束（限制规整路径，提升效率）
  - 相似度评分与最优对齐路径输出
  - 多距离度量支持（欧氏距离、余弦距离、曼哈顿距离）
  - 逐关节偏差分析（识别哪些关节偏差最大）
- **预计工时**: 10-14 小时
- **参考文献支撑**:
  - DTW综述论文：经典DTW(O(n²)), FastDTW(线性), DDTW, SDTW, WDTW, 五阶段DTW
  - MediaPipe论文：流处理中的时间序列同步思想
- **前置依赖**: Phase 1（关键点序列）, Phase 2（标准动作模板）

---

### Phase 4: 深度学习动作分类模块 ✅ 已完成

- **OpenSpec Change**: `dl-action-classifier`（待创建）
- **目录**: `src/models/`
- **核心能力**:
  - CNN 空间特征提取（单帧姿态特征）
  - LSTM/GRU 时序建模（动作序列的时间依赖）
  - CNN-LSTM 混合架构（时空联合特征学习，参考 ASD 综述）
  - 动作分类（识别动作类别，如深蹲、俯卧撑、举臂等）
  - 动作质量评分（在分类基础上评估动作完成度）
  - 模型训练/验证/测试流水线
  - 模型保存与加载
- **预计工时**: 16-20 小时
- **参考文献支撑**:
  - ASD综述：CNN-LSTM时空特征提取、ResNet-50骨干网络、双流网络架构
  - ASD综述Table 5：各特征提取网络优缺点对比
  - ASD综述Table 6：不同分类方法的性能对比
- **前置依赖**: Phase 1（特征提取）, Phase 2（训练数据集）

---

### Phase 5: 动作矫正与反馈模块 ✅ 已完成

- **OpenSpec Change**: `correction-feedback`（待创建）
- **目录**: `src/correction/`
- **核心能力**:
  - 逐关节偏差定位（哪些关节偏离标准最大）
  - 偏差严重程度分级（轻微/中等/严重）
  - 矫正建议生成（自然语言提示，如"左膝弯曲角度不足，请再下蹲10度"）
  - 矫正建议模板库（预定义的常见错误与对应建议）
  - 实时矫正信号（视觉标注 + 文字提示）
  - 动作评分报告生成（一次训练的综合评估）
- **预计工时**: 10-12 小时
- **前置依赖**: Phase 3（DTW偏差分析）, Phase 4（动作分类结果）

---

### Phase 6: 前端界面与实时展示 ✅ 已完成

- **OpenSpec Change**: `web-ui`（待创建）
- **目录**: `src/app/`
- **核心能力**:
  - Streamlit/Gradio Web 应用
  - 实时视频流展示（带骨骼标注）
  - 标准动作选择面板
  - 实时相似度评分仪表盘
  - 矫正建议显示区域
  - 历史记录与训练报告查看
  - 系统配置界面
- **预计工时**: 10-14 小时
- **前置依赖**: Phase 1-5 所有核心模块

---

### Phase 7: 系统集成与优化 ✅ 已完成

- **OpenSpec Change**: `system-integration`（待创建）
- **目录**: 全局
- **核心任务**:
  - 端到端流水线集成（视频输入 → 姿态估计 → 对比/分类 → 矫正反馈 → UI 展示）
  - 实时模式性能优化（目标：≥20FPS 端到端）
  - 离线批处理模式
  - 系统测试（集成测试、性能测试）
  - 部署与演示准备
- **预计工时**: 8-10 小时
- **前置依赖**: Phase 1-6

## 模块依赖关系

```
Phase 1 ──▶ Phase 2 ──▶ Phase 3 ──▶ Phase 5 ──▶ Phase 6 ──▶ Phase 7
   │                        ▲           ▲
   └──────▶ Phase 4 ────────┘───────────┘
```

- Phase 1 是所有后续模块的基础
- Phase 2 是 Phase 3 和 Phase 4 的数据基础
- Phase 3（DTW）和 Phase 4（深度学习）可以并行开发
- Phase 5 需要 Phase 3 和 Phase 4 的结果
- Phase 6 需要所有核心模块就绪
- Phase 7 是最终集成
