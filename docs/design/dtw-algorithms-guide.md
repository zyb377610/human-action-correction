# DTW 算法详解 — 动作对比核心技术

> 本文档是毕业设计「基于深度学习的人体动作矫正系统」的技术补充文档，详细讲解系统中使用的三种 DTW 算法的原理、实现和应用场景。

## 1. 为什么需要 DTW？

### 核心问题

假设用户做了一个"深蹲"动作，标准模板也是"深蹲"。但实际情况中：

- 用户做得**比标准慢**（3 秒 vs 2 秒）
- 用户在中间**停顿**了一下
- 两段序列的**帧数不同**（90 帧 vs 60 帧）

**直接逐帧对比是不行的** —— 用户的第 30 帧可能对应标准的第 20 帧。

**需要一种方法做"时间弹性对齐"**，这就是 DTW 的核心价值。

### 直觉类比

想象两个人在钢琴上弹同一首曲子，一个弹得快、一个弹得慢。DTW 就是找到**"哪个音对哪个音"的最佳匹配**，忽略速度差异，只关注内容是否一致。

---

## 2. 经典 DTW (Dynamic Time Warping)

### 核心思想

> 找到两个序列之间的**最优时间对齐方式**，使得总距离最小。

### 算法步骤

#### Step 1：构建距离矩阵

对每一对帧 `(qᵢ, tⱼ)` 计算距离（如欧氏距离）：

```
用户序列 Q:  q₁  q₂  q₃  q₄  q₅   (5 帧)
模板序列 T:  t₁  t₂  t₃  t₄       (4 帧)

距离矩阵:
          t₁    t₂    t₃    t₄
    q₁ [ 0.1   0.5   0.8   1.2 ]
    q₂ [ 0.4   0.1   0.5   0.9 ]
    q₃ [ 0.7   0.3   0.1   0.4 ]
    q₄ [ 1.0   0.6   0.3   0.1 ]
    q₅ [ 1.3   0.9   0.5   0.2 ]
```

#### Step 2：动态规划填充累积代价矩阵

```
cost[i][j] = dist(qᵢ, tⱼ) + min(
    cost[i-1][j-1],   ← 对角线（一对一匹配）
    cost[i-1][j],     ← 垂直方向（用户多帧 → 模板一帧，用户"慢了"）
    cost[i][j-1]      ← 水平方向（模板多帧 → 用户一帧，用户"快了"）
)
```

#### Step 3：回溯最优路径

从右下角 `cost[N-1][M-1]` 回溯到左上角 `cost[0][0]`，得到最优对齐路径。

### 对齐路径示意

```
用户帧 →   q₁  q₂  q₃  q₄  q₅
            ╲                    
模板帧 →    t₁  t₂  t₃  t₃  t₄
            ↑               ↑
        一对一对齐      q₄ 和 q₃ 都对齐到 t₃
                        (说明用户在这里动作慢了)
```

### Sakoe-Chiba 带宽约束

限制对齐路径不能偏离对角线太远，减少搜索空间：

```
不加约束:                加约束 (window=2):
┌─────────────┐         ┌─────────────┐
│           ╱ │         │     ██╱██   │
│         ╱   │         │   ██╱██     │
│       ╱     │         │ ██╱██       │
│     ╱       │         │██╱██        │
│   ╱         │         │╱██          │
└─────────────┘         └─────────────┘
搜索整个矩阵             只搜索对角线附近的带状区域
(慢但精确)               (快且通常够精确)
```

### 复杂度

- **时间复杂度**: O(N × M)
- **空间复杂度**: O(N × M)
- **本系统中**: 60 帧归一化后，矩阵 60×60 = 3600 格，计算时间 < 50ms

## 3. FastDTW (快速近似 DTW)

### 核心思想

> 不必搜索整个矩阵，用**多分辨率逐步逼近**就够了。

### 三步策略

```
原始序列 (60帧):   ●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●...

Step 1: 粗化 — 逐步降采样到低分辨率
  60帧 → 30帧 → 15帧 → 8帧

Step 2: 在最低分辨率上执行精确 DTW
  ┌────────┐
  │  ╱     │  ← 只搜索 8×8 = 64 个格子，极快
  │╱       │
  └────────┘

Step 3: 投影 + 精化 — 逐层放大，只搜索上一层路径附近
  8帧路径  → 投影到 15帧  → 只搜索路径 ± radius 区域
  15帧路径 → 投影到 30帧  → 只搜索路径 ± radius 区域
  30帧路径 → 投影到 60帧  → 最终结果
```

### 多分辨率图示

```
Level 3 (8帧):    Level 2 (15帧):    Level 1 (30帧):    Level 0 (60帧):
┌──────┐          ┌──────────┐       ┌──────────────┐   ┌──────────────────┐
│ ╱    │  投影→   │ ██╱██    │投影→  │   ██╱██      │→  │     ██╱██        │
│╱     │          │██╱██     │       │  ██╱██       │   │    ██╱██         │
└──────┘          └──────────┘       └──────────────┘   └──────────────────┘
精确 DTW           局部搜索            局部搜索            最终路径
```

### 复杂度

- **时间复杂度**: O(N) — **线性**！相比经典 DTW 的 O(N²) 大幅提升
- **精度**: 论文证明误差 < 5%，在动作矫正场景中完全够用
- **何时使用**: 序列很长（> 300 帧）时切换到 FastDTW

## 4. DDTW (Derivative DTW)

### 核心思想

> 不比较**绝对位置**，比较**变化趋势（形状）**。

### 经典 DTW 的局限

```
标准动作:  手臂高度 = [0.3, 0.5, 0.7, 0.9, 0.7, 0.5, 0.3]  ← 举起再放下
用户动作:  手臂高度 = [0.5, 0.7, 0.9, 1.1, 0.9, 0.7, 0.5]  ← 同样举起放下，但位置偏高

经典 DTW 判定:  "有偏差！每帧都差了 0.2"       ← 基于绝对位置
DDTW 判定:      "完全一样！变化趋势一致"       ← 基于形状
```

### 做法

先对序列求一阶导数（变化率），再对导数序列执行经典 DTW：

```
原始序列:     q₁=0.3   q₂=0.5   q₃=0.7   q₄=0.9   q₅=0.7
                  ↓ 求一阶导数 (相邻帧之差)
导数序列:     d₁=0.2   d₂=0.2   d₃=0.2   d₄=-0.2  d₅=-0.2

然后对导数序列执行经典 DTW
```

### 为什么验证中 DDTW 给出 100% 相似度？

```
标准序列: 每帧关节坐标 = [0.00, 0.03, 0.06, 0.09, ...]  (每帧 +0.03)
用户序列: 每帧关节坐标 = [0.15, 0.18, 0.21, 0.24, ...]  (每帧 +0.03)

一阶导数:
  标准: [0.03, 0.03, 0.03, ...]   ← 变化率
  用户: [0.03, 0.03, 0.03, ...]   ← 变化率完全一样！

DDTW 距离 = 0，相似度 = 100%
```

两个序列只是"平移"了位置，"形状变化"完全一致，所以 DDTW 认为它们相同。**这正是 DDTW 的设计意图**。

### 边界处理

一阶差分会丢失首帧，本系统使用**首帧复制填充**保持序列长度不变：

```python
d[0] = d[1]      # 首帧 = 第二帧的导数
d[-1] = d[-2]    # 末帧 = 倒数第二帧的导数
```

## 5. 三种算法对比总表

| 维度 | 经典 DTW | FastDTW | DDTW |
|------|---------|---------|------|
| **比较对象** | 原始坐标 | 原始坐标 | 坐标的一阶导数（变化率） |
| **时间复杂度** | O(N × M) | O(N) | O(N × M) |
| **精度** | 精确 | 近似（误差 < 5%） | 精确 |
| **敏感于** | 绝对位置差异 | 绝对位置差异 | 形状/趋势差异 |
| **不敏感于** | 时间速度差异 | 时间速度差异 | 位置偏移/体型差异 |
| **适合场景** | 短序列精确比较 | 长序列快速比较 | 忽略体型，关注动作形状 |
| **本系统角色** | **主力算法** | 实时/长序列备选 | 辅助参考 |

## 6. 在本系统中的应用

### 数据流

```
用户动作 PoseSequence          标准模板 PoseSequence
         │                              │
         ▼                              ▼
   preprocess_pipeline()         preprocess_pipeline()
   (插值 → 平滑 → 重采样60帧)   (插值 → 平滑 → 重采样60帧)
         │                              │
         ▼                              ▼
   sequence_to_feature_matrix()  sequence_to_feature_matrix()
   → (60, 99) ndarray           → (60, 99) ndarray
   (33关键点 × xyz坐标)         (33关键点 × xyz坐标)
         │                              │
         └──────────┬───────────────────┘
                    ▼
          compute_dtw()  ← 选择 dtw / fastdtw / ddtw
                    │
                    ▼
         ComparisonResult
         ├── distance       → DTW 总距离
         ├── similarity     → 归一化相似度 (0~100%)
         ├── path           → 最优对齐路径
         └── cost_matrix    → 累积代价矩阵
                    │
                    ▼
         JointDeviationAnalyzer  → 沿对齐路径分析每个关节偏差
                    │
                    ▼
         DeviationReport
         ├── worst_joints   → 偏差最大的 5 个关节
         ├── severity       → 轻微 / 中等 / 严重
         └── frame_deviations → 逐帧偏差曲线
```

### 推荐使用策略

1. **默认使用经典 DTW** — 60 帧归一化后仅 3600 格，< 50ms 完成，精度最高
2. **序列超长时切 FastDTW** — 如分析完整 5 分钟健身视频（> 300 帧）
3. **消除体型差异时参考 DDTW** — 不同身高/体型的用户做同一动作时，DDTW 能更公平地评估动作形状是否正确

### 相似度计算公式

```
normalized_distance = dtw_distance / path_length   （路径归一化，消除长度影响）
similarity = 1 / (1 + normalized_distance)          （映射到 (0, 1] 区间）
```

- similarity = 1.0 (100%): 完全匹配
- similarity → 0: 完全不匹配
- 系统阈值: similarity < 0.3 → 判定为不同动作
